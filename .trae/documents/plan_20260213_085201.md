# 代码提交计划

## 提交内容

### 需要提交的文件
1. **修改的文件**：
   - `Makefile`：添加了代码覆盖率命令
   - `main.go`：添加了端口自动检测功能

2. **新增的文件**：
   - `client/` 目录：客户端SDK实现
   - `start-instances.sh`：多实例启动脚本
   - `status-instances.sh`：多实例状态检查脚本
   - `stop-instances.sh`：多实例停止脚本
   - `.trae/documents/` 目录下的所有文档

### 不需要提交的文件
- 测试数据目录（如 `data/`、`service/data/`、`storage/data/` 等）
- `.idea/` 目录

## 提交步骤

1. **添加文件到git**：
   - `git add Makefile main.go client/ start-instances.sh status-instances.sh stop-instances.sh .trae/documents/`

2. **提交代码**：
   - 使用英文提交描述，包含以下内容：
     - 端口自动检测功能
     - 多实例部署脚本
     - 客户端SDK实现
     - 代码覆盖率命令
     - 相关文档更新

3. **推送到远端**：
   - `git push origin main`

## 预期结果

- 所有修改和新增的文件都被成功提交到git仓库
- 代码被推送到远端仓库
- 提交信息清晰描述了本次提交的内容
